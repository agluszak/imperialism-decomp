```bash

```

### 3) Rebuild (Docker rootless-safe mode)
```bash
docker build --network host -t imperialism-msvc500 -f docker/msvc500/Dockerfile docker/msvc500
docker run --rm --network none \
  -e CMAKE_FLAGS='-DCMAKE_BUILD_TYPE=RelWithDebInfo -DIMPERIALISM_MATCH_FLAGS_CSV=/Oy-,/Ob1' \
  -v "$PWD":/imperialism \
  -v "$PWD/build-msvc500":/build \
  imperialism-msvc500
```

### 4) Detect + compare
```bash
cd build-msvc500
uv run reccmp-project detect --what recompiled
uv run reccmp-reccmp --target IMPERIALISM
```

```bash
uv run python tools/reccmp/progress_stats.py --target IMPERIALISM --build-dir build-msvc500 --detect-recompiled
```

```bash
uv run python tools/reccmp/library_inventory.py --json-out build-msvc500/library_inventory.json
uv run python tools/reccmp/generate_ignore_functions.py --target IMPERIALISM --apply
```


## Known Pitfalls
1. Ghidra decompiler output is generated on-demand; full sync re-decompiles functions.
2. Single huge `stubs.cpp` breaks old MSVC linker/debug limits; use chunked stubs in `src/autogen/stubs/`.
3. Rootless Docker networking may fail with bridge mode; use `--network none` for `docker run`.
4. Two unresolved thunk callees are currently handled by local placeholders in `src/game/thunks.cpp`.
5. Match-flag plumbing accepts both old slash format (`/Oy-/Ob1`) and CSV format (`/Oy-,/Ob1`); use CSV in commands for clarity.

## Current Priorities
1. Keep focused ignore list stable and review weekly.
2. Move from wrapper/thunk/no-op wins into medium-sized game functions.
3. Improve type/prototype fidelity in hotspots before deep body work.
